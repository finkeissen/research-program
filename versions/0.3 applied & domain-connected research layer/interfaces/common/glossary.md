# Glossary (Interfaces)

Dieses Glossar legt Begriffe fest, die in allen Disziplin-Interfaces konsistent verwendet werden.

## Claim
Eine Aussage, die einen Wahrheitsanspruch oder Gültigkeitsanspruch trägt.
- hat eine Form (natürlichsprachlich, formal, operationalisiert)
- kann Bedingungen/Gültigkeitsbereich haben

## Definition
Eine Festlegung eines Ausdrucks innerhalb eines Systems/Modells.
- nicht automatisch empirisch „wahr“
- kann mehrere Ebenen haben (sprachlich, formal, operational)

## Assumption (Annahme)
Ein Satz/Parameter/Regel, der für eine Ableitung oder Modellierung vorausgesetzt wird.

## Evidence (Evidenz)
Stützung eines Claims durch Daten, Messungen, Ableitungen oder verlässliche Referenzen.
- kann empirisch oder formal sein
- hat Qualität (Stärke, Unsicherheit, Reproduzierbarkeit)

## Argument / Justification (Begründung)
Struktur, die Claims aus Annahmen/Definitionen/Regeln ableitet oder plausibilisiert.
- formal (Logik/Mathe)
- operational (CS: Tests/Verifikation)
- empirisch (Physik: Messprotokoll + Auswertung)

## Model (Modell)
Ein formales oder semi-formales Objekt, das bestimmte Aspekte eines Systems beschreibt.
- hat Gültigkeitsbereich
- erzeugt Vorhersagen/Constraints

## System / World
- System: Gegenstand innerhalb eines Modells (modell-interne Entität)
- World: empirische Realität (mess-/beobachtungsgebunden)

## Boundary (Grenze)
Explizite Markierung, was ein Interface leistet und was nicht.
- verhindert Kategorienfehler (z. B. Beweis ↔ Messung)

## Translation (Übersetzung)
Mapping zwischen Repräsentationsformen:
- Sprache → formal
- formal → algorithmisch
- formal/algorithmisch → messbar/operationalisiert
