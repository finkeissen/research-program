# Computer Science Boundary

## Nicht-Ziele
- Keine Garantie für reale Laufzeit auf physischer Hardware.
- Keine Aussage über empirische Angemessenheit von Modellen.
- Keine normative Entscheidung über „richtige“ Approximationen.

## Typische Kategorienfehler
- Berechenbar = praktisch machbar
- Algorithmus existiert = Algorithmus implementiert
- Test bestanden = korrekt bewiesen

## Verhältnis zu anderen Interfaces
- nutzt Mathematik als formale Basis
- liefert operative Strukturen für mms
- grenzt sich klar von Physik ab (keine Messrealität)

## Abbruchkriterien
- Unklare Eingabedomäne
- Nicht spezifizierte Approximationen
- Implizite Ressourcenannahmen
