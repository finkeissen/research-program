# Mathematics Boundary

## Nicht-Ziele
- Keine empirische Interpretation von Variablen.
- Keine Entscheidung über „Realitätsnähe“ von Modellen.
- Keine algorithmische Effizienzbewertung (→ CS).

## Typische Kategorienfehler
- Modell = Realität
- Existenzbeweis = Konstruktionsanweisung
- Grenzwert = erreichbarer Wert

## Verhältnis zu anderen Interfaces
- baut auf `logic/` auf (Signaturen, Ableitbarkeit)
- liefert Input für:
  - `computer-science/` (Algorithmen, Approximation)
  - `physics/` (Modelle + Gültigkeitsbereiche)

## Abbruchkriterien
- Unklare oder widersprüchliche Definitionen
- Nicht überprüfbare Beweislücken
- Annahmen nicht explizit machbar
