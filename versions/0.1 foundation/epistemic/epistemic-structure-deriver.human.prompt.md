```txt
# ============================================================
# Epistemischer Derivations-Prompt (menschenlesbar)
# Bindet explizit an den Bootloader an
# Vor-technisch, vor-formal, vor-numerisch
# ============================================================

# PRÄAMBEL
# Dieser Prompt setzt den BOOTLOADER als verbindliche Grundlage voraus.
# Alle folgenden Aussagen unterliegen dessen Regeln:
# - Alles ist ein PROBLEM.
# - PROBLEME existieren nur durch Notwendigkeiten (Zwang),
#   aus denen zwingende Konsequenzen folgen.
# - Struktur entsteht durch Ableitung, nicht durch Setzung.

# AUFGABE
# Beschreibe die Struktur der Welt so, dass sie aus Notwendigkeiten
# hervorgeht und nicht aus Konventionen, Disziplinen oder Meinungen.
# Leite dabei vier Ebenen ab, ohne neue Ontologien einzuführen:
# 1. Domänen
# 2. Subdomänen
# 3. Themengebiete / Fachgebiete
# 4. Atomare Probleme
# Diese Ebenen unterscheiden sich nicht im Schema,
# sondern ausschließlich im Abstraktionsgrad.

# ------------------------------------------------------------
# DOMÄNEN
# ------------------------------------------------------------
# Definition:
# Eine DOMÄNE ist ein irreduzibler Typ von Notwendigkeit (Zwang),
# der nicht sinnvoll auf einen anderen Typ von Notwendigkeit
# zurückgeführt werden kann.

# Gültigkeitskriterien für eine Domäne:
# - Irreduzibilität:
#   Der zugrunde liegende Zwang ist nicht vollständig erklärbar
#   durch einen anderen Zwangstyp.
# - Explanatorische Notwendigkeit:
#   Es existieren Sachverhalte, die ohne diese Domäne
#   nicht sinnvoll beschreibbar sind.
# - Trennschärfe:
#   Der Kern-Zwang unterscheidet sich qualitativ
#   von anderen Domänen.
# - Stabilität:
#   Die Domäne bleibt über Kontexte, Zeiten und Fachgebiete hinweg relevant.
# - Minimalität:
#   Das Entfernen der Domäne führt zu einem Verlust
#   an erklärender oder entscheidungsrelevanter Kraft.

# Regel:
# Leite genau so viele Domänen ab, wie notwendig und so wenige wie möglich.
# Die Anzahl ergibt sich aus der Struktur der Welt,
# nicht aus einer vorgegebenen Zahl.

# ------------------------------------------------------------
# SUBDOMÄNEN
# ------------------------------------------------------------
# Definition:
# Eine SUBDOMÄNE ist eine kontextuelle Einschränkung einer Domäne,
# bei der der Typ der Notwendigkeit gleich bleibt.

# Subdomänen entstehen durch:
# - Kontext
# - Reichweite
# - Zeitlichkeit
# - Verantwortungsrahmen

# Abbruchregel:
# Unterteile nur so weit, wie dadurch neue zwingende Konsequenzen
# ableitbar werden.
# Sobald eine weitere Unterteilung keinen zusätzlichen
# Erkenntnis- oder Entscheidungsgewinn erzeugt,
# ist sie epistemisch unzulässig.

# ------------------------------------------------------------
# THEMENGEBIETE / FACHGEBIETE
# ------------------------------------------------------------
# Einordnung:
# Themen- und Fachgebiete sind keine eigenständige epistemische Ebene.
# Sie sind Projektionen (Sichten) auf Gruppen von Problemen.

# Eigenschaften:
# - historisch
# - institutionell
# - praxisgetrieben
# - veränderlich

# Regel:
# Themengebiete dürfen Struktur sichtbar machen,
# aber keine neue Struktur einführen.
# Sie leiten sich aus Praxis ab, nicht aus Notwendigkeit.

# ------------------------------------------------------------
# ATOMARE PROBLEME
# ------------------------------------------------------------
# Definition:
# Ein ATOMARES PROBLEM ist die kleinste noch sinnvolle Beschreibung
# eines Sachverhalts, bei der eine weitere Zerlegung
# Kausalität, Verantwortung oder Entscheidbarkeit zerstören würde.

# Kriterien:
# - Unteilbarkeit:
#   Weitere Aufspaltung führt zu Bedeutungsverlust.
# - Eindeutiger Zwang:
#   Genau ein dominanter Typ von Notwendigkeit wirkt im Kern.
# - Entscheidungsfähigkeit:
#   Aus der Beschreibung lassen sich zwingende Konsequenzen ableiten.
# - Kontextualität:
#   Atomarität gilt immer relativ zu Domäne und Kontext,
#   niemals absolut.

# ------------------------------------------------------------
# FIXPUNKTREGEL
# ------------------------------------------------------------
# Domänen bestimmen, wann ein Problem als atomar gilt.
# Atomare Probleme prüfen, ob eine Domäne tatsächlich notwendig ist.
# Beide legitimieren sich gegenseitig.
# Stabilität entsteht nur dort, wo dieser Fixpunkt erreicht ist.

# ------------------------------------------------------------
# STOP-REGEL (ENTSCHEIDEND)
# ------------------------------------------------------------
# Jede weitere Differenzierung ist zu unterlassen,
# sobald sie keine neuen zwingenden Konsequenzen mehr erzeugt.
# Keine Konsequenz → keine neue Struktur.

# ------------------------------------------------------------
# META-REGEL
# ------------------------------------------------------------
# Arbeite ohne:
# - numerische Zielvorgaben
# - Disziplinlisten
# - narrative Rechtfertigungen
#
# Ziel ist nicht, Struktur zu behaupten,
# sondern sie aus epistemischer Notwendigkeit sichtbar zu machen.

# ============================================================
# ENDE DES PROMPTS
# ============================================================
```
