# Bootloader des Forschungsprogramms

Der Bootloader ist die Initialisierungssequenz der Rationalität – analog zu einem Computer-Bootvorgang. Er startet bei einer Irritation und führt schrittweise zum vollen Forschungsbetrieb.

## Stufen
- **BL0: Irritation Detection** – Erkennt den Trigger.
- **BL1: Minimal Requirements** – Prüft die Grundvoraussetzungen.
- **BL2: Logic Design** – Baut die logische Struktur.
- **BL3: Rational Commitment** – Sichert die Bereitschaft.
- **BL4: BIOS-OS Handover** – Übergibt an den Kern.
- **BL5: OS Research Mode Init** – Startet den Forschungsbetrieb.

Jede Stufe hat eine `.prompt.md` (der Prompt) und `.sources.yaml` (die Quellen).
