# Validation (Cross-Discipline)

Validierung heißt hier: „ist dieses Artefakt *akzeptabel* für die nächste Stufe?“

## Minimalchecks (für alle Interfaces)
- Vollständige Angaben: Claim + Scope + Annahmen/Abhängigkeiten
- Keine Vermischung: formale Ableitung wird nicht als Messbeleg ausgegeben (und umgekehrt)
- Boundary geprüft: Artefakt bleibt im Zuständigkeitsbereich der Disziplin

## Disziplin-spezifische Checks (Platzhalter)
- logic/: Konsistenzmarker, Ableitungsstruktur
- mathematics/: Definitionskohärenz, Lemma-Kette
- computer-science/: Ausführbarkeit/Komplexitätsannahmen, Testbarkeit
- physics/: Messprotokoll, Einheiten, Unsicherheit, Kalibrierung
